# MiniApp silent identity and account merge

The MiniApp establishes a non-blocking MiniIdentity from the current Mini Program OpenID at cold start, represented by a Better Auth anonymous user and a channel-specific Bearer session; public browsing remains available when authentication fails, and inactive unbound identities without business data may be removed after 90 days. A MiniIdentity may exist independently, but binding is a one-to-one, irreversible IdentityMerge into a SignedInViewer: Bangumi X Web creates a short-lived single-use challenge from the active signed-in session without another authentication round trip, the MiniApp scans and confirms the target profile, and existing Mini data moves to the SignedInViewer while the OpenID becomes its login credential. Unbinding requires recent Easy Auth reauthentication, revokes only Mini sessions, and removes that credential without splitting merged data back out, so the same OpenID later creates a new empty MiniIdentity; this avoids parallel identity ownership and synthetic permanent profiles while preserving silent MiniApp login.
